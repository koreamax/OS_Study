**최민경**

1. ALU의 연산 결과인 플래그 값이 왜 제어장치로 전달되어야 하나요?
<details>
  <summary>정답</summary>
  제어장치가 플래그 값을 받는 이유는 연산 결과에 따라 CPU의 실행 흐름을 동적으로 제어하기 위해서입니다. 컴퓨터의 명령어 중에는 Conditional Jump 같은 명령어도 존재하는데, 이런 명령어는 연산 직후의 상태를 ALU만 알고 있습니다. 따라서 제어장치는 ALU가 전달한 플래그 값을 기반으로 다음에 실행할 명령어의 주소를 결정하거나 예외 상황 처리 여부를 판단합니다.
</details>

2.상대 주소 지정 방식, 베이스 레지스터 주소 지정 방식, 스택 주소 지정 방식은 각각 어떤 용도로 사용되나요?
<details>
  <summary>정답</summary>
  상대 주소 방식은 PC를 기준으로 하여 프로그램 내부의 분기 명령어를 효율적으로 처리하기 위해 사용합니다. 베이스 레지스터 방식은 프로그램의 시작 주소를 기준으로 삼아 메모리 재배치를 용이하게 하는 것이 주된 목적입니다. 마지막으로 스택 주소 방식은 주소를 명시하지 않고 스택 포인터를 이용해 함수 호출이나 인터럽트 처리 시 임시 데이터를 빠르게 관리하기 위해 사용합니다.
</details>



**방혜윤**

1-1. 레지스터 중 메모리 주소 레지스터와 메모리 버퍼 레지스터를 설명해주세요
<details>
  <summary>정답</summary>
  메모리 주소 레지스터는 CPU가 접근하려는 메모리의 주소를 저장하는 레지스터입니다.

  메모리 버퍼 레지스터는 메모리와 주고받는 데이터나 명령어 자체를 저장하는 레지스터입니다.
</details>

1-2. 메모리에 저장된 프로그램을 실행하는 과정에서 레지스터들의 값이 어떻게 변화하는지 설명해주세요
<details>
  <summary>정답</summary>
  PC → 메모리 주소 레지스터: PC에 저장된 다음 실행할 주소가 메모리 주소 레지스터로 복사됩니다.
  
  메모리 주소 레지스터→ 메모리: 메모리 주소 레지스터의 주소값이 주소 버스를 통해 메모리로 전달되고, 제어 버스로 '읽기' 신호가 보내집니다.
  
  메모리 → 메모리 버퍼 레지스터: 메모리 해당 주소의 명령어가 데이터 버스를 통해 메모리 버퍼 레지스터로 들어옵니다. 동시에 PC 값은 증가합니다.
  
  메모리 버퍼 레지스터→ 명령어 레지스터: 메모리 버퍼 레지스터에 담긴 명령어가 명령어 레지스터로 이동하여 해석될 준비를 마칩니다.
</details>

2-1. 주소 지정 방식 중 변위 주소 지정 방식을 설명해주세요
<details>
  <summary>정답</summary>
  명령어의 오퍼랜드 필드에 담긴 값과 특정 레지스터의 값을 더하여 실제 데이터가 위치한 유효 주소를 구하는 방식입니다. 명령어의 제한된 비트 수로도 넓은 범위의 메모리 주소를 지정할 수 있게 해줍니다.
</details>

2-2. 변위 주소 지정 방식은 상대 주소 지정 방식과 베이스 레지스터 주소 지정 방식으로 나뉘는데 각각을 설명해주세요
<details>
  <summary>정답</summary>
  상대 주소 지정 방식: 오퍼랜드와 프로그램 카운터의 값을 더합니다. 현재 실행 중인 위치를 기준으로 앞뒤로 이동할 때 사용하며, 주로 분기문 처리에 유리합니다.

  베이스 레지스터 주소 지정 방식: 오퍼랜드와 베이스 레지스터의 값을 더합니다. 프로그램이 메모리의 어디에 로드되든 기준점으로부터의 거리만 알면 되므로 프로그램 재배치에 매우 유용합니다
</details>

**이민형**

1-1. 변위 주소 지정 방식에서 명령어 필드는 무엇으로 구성되나요?
<details>
  <summary>정답</summary>
  연산코드(opcode) + 레지스터 필드 + 오퍼랜드
</details>

1-2. 시스템 버스에서 사용되는 버스의 종류 3가지와 각각의 역할을 설명하시오.
<details>
  <summary>정답</summary>
  데이터 버스: 실제 데이터를 전달하는 통로<br>
  주소 버스: CPU가 접근할 메모리/장치의 주소 전달<br>
  제어 버스: 읽기/쓰기, 인터럽트 등 제어 신호 전달
</details>

2-1. 주요 플래그 레지스터 4가지와, 각 플래그 값이 1일 때의 의미를 설명하시오.
<details>
  <summary>정답</summary>
  부호 플래그: 1이면 결과가 음수<br>
  캐리 플래그: 1이면 자리올림/빌림 발생<br>
  제로 플래그: 1이면 결과가 0<br>
  오버플로우 플래그: 1이면 오버플로우 발생
</details>

2-2. 프로그램 카운터(PC)가 단순히 +1 증가하지 않는 경우를 설명하시오.
<details>
  <summary>정답</summary>
  분기 명령어(jump, branch, call, return) 실행 시 → 특정 주소로 이동<br>
  인터럽트 발생 시 → 인터럽트 처리 루틴 주소로 이동
</details>
