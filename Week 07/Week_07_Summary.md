# Chapter 06. 메모리와 캐시 메모리

CPU에 이어 컴퓨터의 핵심 부품인 **메모리**를 다룬다. 여기서 메모리는 주기억장치(메인 메모리), 그중에서도 **RAM**을 지칭한다. 마지막으로 CPU와 메모리 사이의 속도 차를 줄이는 **캐시 메모리**도 살펴본다.

## 06-1 RAM의 특징과 종류

### RAM의 특징

- CPU는 RAM(메모리)으로부터 명령어와 데이터를 가져와 실행한다. RAM은 **실행할 대상**을 저장한다.
- RAM은 **전원이 꺼지면 저장된 내용을 잃는다.** 그래서 전원이 꺼져도 내용을 보관할 수 있는 대용량 저장 장치인 **보조기억장치**(SSD, 하드디스크, USB, CD-ROM 등)가 필요하다.

| 구분 | 특성 | 저장 대상 |
| --- | --- | --- |
| **휘발성 저장 장치** | 전원이 꺼지면 내용을 잃음 | RAM → **실행할** 대상 |
| **비휘발성 저장 장치** | 전원이 꺼져도 내용을 유지 | 보조기억장치 → **보관할** 대상 |

### RAM의 용량과 성능

RAM의 용량은 컴퓨터 성능에 큰 영향을 준다. ("다다익램"이라는 말이 있을 만큼 중요한 선정 기준이다.)

- RAM이 작으면, 보조기억장치에서 프로그램을 그때그때 복사해 와야 해서 여러 프로그램을 동시에 실행하기 불리하다.
- RAM이 크면, 필요한 프로그램이 이미 RAM에 있어 보조기억장치를 오갈 필요가 줄어든다.

> **책상 비유** — 보조기억장치 = 책장(많이 보관), RAM = 책상(읽을 책을 올려둠). 책상이 크면 여러 책을 미리 올려두고 읽을 수 있어 책장을 자주 오갈 필요가 없다. 즉 동시에 여러 프로그램을 실행하기 유리하다.

### RAM의 종류

RAM은 크게 **DRAM, SRAM, SDRAM, DDR SDRAM** 네 종류로 나뉜다.

#### DRAM (Dynamic RAM)

- **동적(Dynamic)**인 RAM. 전원이 연결되어 있어도 **시간이 지나면 저장된 데이터가 점점 사라진다.**
- 데이터 소멸을 막기 위해 주기적으로 다시 저장하는 **재활성화(refresh)**가 필요하다.
- 소비전력이 낮고 가격이 저렴하며 **집적도가 높아** 대용량 설계에 유리하다. → **메모리(주기억장치)로 사용**된다.

#### SRAM (Static RAM)

- **정적(Static)**인 RAM. 전원이 연결되어 있으면 데이터가 사라지지 않는다. (단, 전원을 빼면 사라지므로 **휘발성**이다.)
- 재활성화가 필요 없고, DRAM보다 **빠르다.**
- 소비전력·가격이 높고 집적도가 낮아 대용량 설계가 어렵다. → 대용량은 아니지만 빠르게 동작해야 하는 **캐시 메모리**에 사용된다.

**DRAM vs SRAM**

| 구분 | DRAM | SRAM |
| --- | --- | --- |
| 재충전(재활성화) | 필요 | 불필요 |
| 속도 | 상대적으로 느림 | 상대적으로 빠름 |
| 가격 | 저렴 | 비쌈 |
| 집적도 | 높음 | 낮음 |
| 소비전력 | 적음 | 많음 |
| 용도 | 메모리(주기억장치) | 캐시 메모리 |

#### SDRAM (Synchronous DRAM)

**클럭 신호에 동기화된(Synchronous)** 발전된 형태의 DRAM이다. (이름이 비슷하지만 SRAM과는 관계없다.) 클럭 신호에 맞춰 한 번에 하나씩 데이터를 주고받는다. 이런 형태를 **SDR(Single Data Rate) SDRAM**이라고도 한다.

#### DDR SDRAM (Double Data Rate SDRAM)

데이터 전송률을 2배로 늘린(대역폭을 넓힌) SDRAM으로, **현재 가장 대중적으로 사용되는** 메모리다.

> **도로 비유** — 대역폭은 데이터가 오가는 길의 너비다. SDR SDRAM이 1차선 도로라면, DDR는 2차선처럼 한 번에 더 많은 데이터를 더 빠르게 주고받는다.

| 종류 | 대역폭 (SDR 기준) |
| --- | --- |
| SDR SDRAM | ×1 |
| DDR SDRAM | ×2 |
| DDR2 SDRAM | ×4 |
| DDR3 SDRAM | ×8 |
| DDR4 SDRAM | ×16 |

> 각 세대는 이전 세대 대비 대역폭이 2배씩 넓어진다. 현재 가장 대중적인 것은 **DDR4 SDRAM**이다.

## 06-2 메모리의 주소 공간

메모리의 주소 체계는 **논리 주소**와 **물리 주소** 두 가지로 나뉜다.

### 물리 주소와 논리 주소

CPU와 실행 중인 프로그램은 메모리 몇 번지에 무엇이 저장되어 있는지 전부 알 수 없다. 메모리에 저장된 값은 시시때때로 변하기 때문이다. (새 프로그램이 적재되고, 끝난 프로그램은 삭제되며, 같은 프로그램도 실행할 때마다 적재되는 주소가 달라진다.) 이를 극복하기 위해 주소 체계를 둘로 나눈다.

| 주소 | 관점 | 설명 |
| --- | --- | --- |
| **물리 주소(physical address)** | 메모리 입장 | 정보가 실제로 저장된 하드웨어상의 주소. 겹치는 번지가 없음 |
| **논리 주소(logical address)** | CPU·실행 중인 프로그램 입장 | 각 프로그램에 **0번지부터** 부여한 주소. 프로그램마다 같은 번지가 중복될 수 있음 |

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/c6a82dac-6531-45ec-92d1-9759801231f1" />


### 물리 주소와 논리 주소의 변환

CPU가 이해하고 실행하는 주소는 모두 **논리 주소**다. 하지만 메모리와 실제로 상호작용하려면 논리 주소를 물리 주소로 변환해야 한다. 이 변환은 CPU와 메모리 사이의 **MMU(메모리 관리 장치, Memory Management Unit)**가 담당한다.

MMU는 **논리 주소 + 베이스 레지스터 값 = 물리 주소** 방식으로 변환한다.

- **베이스 레지스터**: 프로그램의 시작 주소(물리 주소상의 기준 주소)를 저장한다.
- **논리 주소**: 그 시작 주소로부터 얼마나 떨어져 있는지를 나타낸다.

> 예) 프로그램 A가 물리 주소 15,000번지부터 저장됨 → 베이스 레지스터 = 15,000. "논리 주소 100번지 삭제"는 기준 주소로부터 100 떨어진 곳, 즉 **물리 주소 15,100번지**를 삭제하라는 의미가 된다.

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/2b42e1a0-e130-4402-9279-68bd5850ba1a" />


### 메모리 보호 기법

논리 주소는 0번지부터 시작하므로, 잘못된 명령어가 **다른 프로그램의 영역을 침범**할 수 있다. (예: 논리 주소 범위가 0~99인 프로그램이 1,500번지에 접근하면, 변환된 물리 주소가 다른 프로그램의 영역을 침범한다.)

이를 막기 위해 **한계 레지스터(리미트 레지스터, limit register)**를 사용한다.

- **베이스 레지스터**: 프로그램의 가장 작은 물리 주소(시작 주소) 저장
- **한계 레지스터**: 논리 주소의 **최대 크기** 저장

따라서 한 프로그램의 정상적인 물리 주소 범위는 다음과 같다.

```
베이스 레지스터 값  ≤  물리 주소  <  베이스 레지스터 값 + 한계 레지스터 값
```

CPU는 명령어가 **한계 레지스터 값보다 큰 논리 주소**에 접근하려 하면 그 실행을 막아, 다른 프로그램 영역 침범을 방지한다.

> 예) 베이스 레지스터 100, 한계 레지스터 150 → 이 프로그램은 물리 주소 100번지 이상 250번지 미만에만 접근할 수 있다.

## 06-3 캐시 메모리

CPU가 메모리에 접근하는 속도는 CPU의 연산 속도보다 압도적으로 느리다. 이 격차를 줄이기 위한 저장 장치가 캐시 메모리다.

### 저장 장치 계층 구조

저장 장치 계층 구조(memory hierarchy)는 다음 두 명제로 요약된다.

1. CPU와 **가까운** 저장 장치는 일반적으로 속도가 **빠르고**, 멀리 있는 저장 장치는 느리다.
2. 속도가 빠른 저장 장치는 일반적으로 용량이 **작고 가격이 비싸다.**

즉, 낮은 가격의 대용량을 원하면 느린 속도를, 빠른 속도를 원하면 작은 용량과 비싼 가격을 감수해야 한다. 컴퓨터는 각 저장 장치를 장단점에 맞게 **계층적으로** 배치해 사용한다.

```
   빠름 / 작음 / 비쌈
        ▲
   ┌─────────────┐
   │  레지스터       │  (CPU 내부)
   ├─────────────┤
   │  캐시 메모리     │  (SRAM 기반)
   ├─────────────┤
   │  메모리(RAM)    │  (DRAM 기반)
   ├─────────────┤
   │  보조기억장치     │  (SSD, HDD 등)
   └─────────────┘
        ▼
   느림 / 큼 / 저렴
```

### 캐시 메모리

**캐시 메모리**는 CPU와 메모리 사이에 위치한, 레지스터보다 용량이 크고 메모리보다 작으며 메모리보다 빠른 **SRAM 기반** 저장 장치다. CPU의 연산 속도와 메모리 접근 속도의 차이를 줄이기 위해, **메모리에서 사용할 일부 데이터를 미리 가져와 저장**한다.

> **마트/편의점 비유** — 메모리는 물건은 많지만 멀리 있어 왕복이 오래 걸리는 대형마트, 캐시 메모리는 물건은 적어도 가까운 편의점. 원하는 물건이 편의점에 있으면 마트까지 갈 필요가 없다.

#### 계층적 캐시 메모리 (L1, L2, L3)

캐시 메모리는 하나가 아니라 계층적으로 구성된다.

| 캐시 | 용량 | 속도 | 위치 |
| --- | --- | --- | --- |
| **L1 캐시** | 가장 작음 | 가장 빠름 | 코어 내부 |
| **L2 캐시** | L1 < L2 < L3 | 중간 | 코어 내부 |
| **L3 캐시** | 가장 큼(캐시 중) | 가장 느림(캐시 중) | 코어 외부(여러 코어가 공유하기도 함) |

> **참고**
> - **분리형 캐시** — L1 캐시를 더 빠르게 하려고 명령어 전용(L1I)과 데이터 전용(L1D)으로 나누기도 한다.
> - **멀티코어의 캐시 일관성** — 코어마다 별도의 캐시를 가지면 각 캐시의 내용이 달라질 수 있어, 모든 캐시가 동일한 내용을 갖도록 동기화하는 **캐시 일관성**이 중요한 문제가 된다.

### 참조 지역성의 원리

캐시 메모리는 용량이 작아 메모리의 일부만 저장할 수 있다. 따라서 **CPU가 자주 사용할 법한 데이터를 예측**해 저장해야 한다.

| 용어 | 의미 |
| --- | --- |
| **캐시 히트(cache hit)** | 예측이 맞아, 캐시에 가져온 데이터를 CPU가 실제로 사용한 경우 (성능 향상) |
| **캐시 미스(cache miss)** | 예측이 빗나가, CPU가 결국 메모리에 접근한 경우 (성능 저하) |
| **캐시 적중률** | 캐시 히트 ÷ (캐시 히트 + 캐시 미스). 높을수록 성능이 좋음 |

> 요즘 CPU의 캐시 적중률은 보통 80% 이상, 90%를 넘기는 경우도 많다.

CPU가 사용할 데이터를 예측하는 원칙이 **참조 지역성의 원리(locality of reference)**다. 두 가지 경향에 기반한다.

| 경향 | 설명 |
| --- | --- |
| **시간 지역성(temporal locality)** | CPU는 **최근에 접근했던** 메모리 공간에 다시 접근하는 경향이 있다. (예: 변수는 한 번 만들면 여러 번 참조됨) |
| **공간 지역성(spatial locality)** | CPU는 접근한 메모리 공간의 **근처**에 접근하는 경향이 있다. (예: 관련된 기능·데이터가 한데 모여 저장됨 — 입력 기능을 쓸 땐 입력 기능 근처를 자주 참조) |

캐시 메모리는 이 두 경향을 바탕으로 자주 참조될 데이터를 미리 가져와 저장함으로써 캐시 적중률을 높이고 성능을 향상시킨다.
