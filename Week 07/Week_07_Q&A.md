**최민경**

1-1. RAM, ROM의 차이를 설명해주세요
<details>
  <summary>정답</summary>
  RAM은 휘발성 특징을 가지며 실행 중인 프로그램/데이터의 임시 저장소 역할을 합니다. 속도는 빠른 편입니다. ROM은 비휘발성 특징을 가지며 읽기 전용입니다. 부팅 시스템, 펌웨어 저장에 사용되며 보통 RAM보다 느립니다.
</details>

1-2. RAM 중에서 대용량일 필요는 없고, 속도는 빠르면 좋을 때 사용할 수 있는 RAM은 무엇일까요?
<details> 
  <summary>정답</summary>
  SRAM. 가장 빠르고 용량이 작으며 캐시 메모리에서 사용합니다.
</details>

1-3. 속도도 빠르면서 용량도 커서 현대 컴퓨터에서 메인 메모리로 사용하는 램은?
<details> 
  <summary>정답</summary>
  DDR. 속도가 빠르며 용량도 커서 현대 컴퓨터에서 메인 메모리로 사용합니다.
</details>

2-1. CPU와 실행 중인 프로그램에서 왜 물리주소를 쓰지않고 논리구조를 쓸까요?
<details>
  <summary>정답</summary>
  메모리는 시시각각 저장된 위치가 변하기 때문에 CPU는 메모리 몇번지에 데이터가 저장되어있는지 알 수 없다. 그래서 논리구조를 쓴다.
</details>

2-2. cpu에서 주소가 물리 주소로 변환될 때 구조를 설명해주세요.
<details>
  <summary>정답</summary>
  CPU → 논리주소버스 → MMU → 물리주소버스 → RAM 구조로 변환됩니다.
</details>



**방혜윤**

1-1. 논리 주소와 물리 주소의 차이를 설명해주세요.
<details>
  <summary>정답</summary>
  물리 주소는 메모리가 사용하는 주소로 정보가 실제로 저장된 하드웨어상의 주소입니다.<br>
  논리 주소는 CPU와 실행 중인 프로그램이 사용하는 주소로 실행 중인 프로그램 각각에 부여된 0번지부터 시작되는 주소입니다.<br>
  CPU는 논리 주소를 사용하고 실제 메모리 접근 시에는 물리 주소로 변환하여 접근합니다.
</details>

1-2. 메모리 접근할 때 다른 프로그램의 메모리 영역을 침범하는 경우를 설명하고 이를 방지하는 방법을 설명해주세요.
<details>
  <summary>정답</summary>
  프로그램이 자신의 주소 공간 범위를 벗어난 메모리에 접근하려고 하면 다른 프로그램이나 운영체제의 영역을 침범할 수 있습니다.<br>
  이를 방지하기 위해 한계 레지스터를 사용합니다.<br>
  CPU는 메모리 접근 전에 논리 주소가 한계 레지스터 범위 내에 있는지 검사하고, 범위를 벗어나면 트랩 인터럽트를 발생시켜 프로그램 실행을 중단합니다.
</details>

2-1. 캐시 메모리에 대해 설명해주세요
<details>
  <summary>정답</summary>
  캐시 메모리는 CPU와 메모리 사이에 위치하는 저장 장치입니다.<br>
  CPU가 자주 사용할 것으로 예상되는 데이터를 미리 저장하여 메모리 접근 시간을 줄이는 역할을 합니다.<br>
  일반적으로 SRAM으로 구현되며 CPU의 연산 속도와 메모리 접근 속도 차이를 줄이기 위해 사용됩니다.
</details>

2-2. 분리형 캐시의 장점을 설명해주세요
<details>
  <summary>정답</summary>
  분리형 캐시는 L1 캐시를 명령어를 저장하는 캐시 L1I와 데이터를 저장하는 캐시 L1D를 분리한 구조입니다.<br>
  이를 통해 CPU가 명령어를 가져오는 작업과 데이터를 읽는 작업을 동시에 수행할 수 있어 캐시 접근 경쟁을 줄이고 성능을 향상시킬 수 있습니다.
</details>

**이민형**

1-1. RAM 용량이 클 때의 장점을 CPU와 보조기억장치 관점에서 설명하세요.
<details>
  <summary>정답</summary>
  CPU 관점에서는 더 많은 데이터와 프로세스를 메모리에 상주시킬 수 있어, CPU가 보조기억장치까지 내려가지 않고 빠르게 데이터를 가져올 수 있습니다.<br>
  보조기억장치 관점에서는 디스크 I/O가 줄어들어 프로세스가 디스크 응답을 기다리며 블로킹되는 상황이 줄고, 보조기억장치가 전체 시스템의 병목이 되는 정도가 완화됩니다.
</details>

1-2. RAM 용량이 클 때 그에 따른 트레이드오프는?
<details>
  <summary>정답</summary>
  가격이 비싸지고, 전력 소모가 증가하며, 실제로 사용하지 않으면 그만큼 낭비가 크고, <br>
  관리해야 할 메모리 영역이 늘어나 메모리 관리 부담이 커집니다.
</details>

2-1. Redis는 일반적인 캐시처럼 SRAM이 아닌 DRAM 위에서 동작합니다. 그 이유는?
<details>
  <summary>정답</summary>
  SRAM은 빠르지만 용량이 작고 가격이 매우 비싸기 때문에, Redis가 다루는 데이터 규모를 올리기에는 부적합합니다.<br>
  또한 Redis가 비교 대상으로 삼는 것은 L1/L2/L3 같은 CPU 캐시가 아니라 DB나 SSD입니다. 즉 Redis는 CPU 캐시보다는 느리지만 DB/SSD보다는 훨씬 빠른 계층이라 캐시라고 부를 수 있습니다.
</details>

2-2. Redis에서 hit ratio(캐시 적중률)가 높을 때 생기는 트레이드오프는?
<details>
  <summary>정답</summary>
  hit ratio를 높이기 위해 TTL을 길게 잡으면 오래된 데이터가 캐시에 더 오래 머무릅니다.<br>
  이 경우 원본 데이터가 수정·변경되었음에도 캐시에서는 갱신 전의 오래된 데이터를 반환하는 상황이 발생할 수 있습니다. <br>
  즉 적중률(성능)과 데이터 최신성(정합성) 사이에 트레이드오프가 생깁니다.
</details>
