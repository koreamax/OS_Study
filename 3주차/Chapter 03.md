1-1. GCC 컴파일러를 사용하여 C 소스 파일(.c)을 실행 파일로 변환하는 명령어를 작성하시오.
<details>
  <summary>정답</summary>
  gcc file.c -o file → file이라는 이름의 실행파일 생성<br> gcc file.c → 기본적으로 a.out이라는 실행파일 생성
</details>

1-2. 리눅스에서 Makefile을 이용하여 컴파일과 링크 과정을 자동화할 때 사용하는 명령어는 무엇인가요?
<details>
  <summary>정답</summary>
  make → Makefile에 정의된 규칙을 기반으로 변경된 파일만 다시 빌드하여 컴파일과 링크를 자동으로 수행
</details>

2-1. 컴파일러와 인터프리터의 차이를 설명해주세요.
<details>
  <summary>정답</summary>
  컴파일러 → 전체 코드를 한 번에 번역하여 실행<br> 인터프리터 → 한 줄씩 번역하며 실행
</details>

2-2. 컴파일과 링커의 차이를 설명해주세요.
<details>
  <summary>정답</summary>
  컴파일 → 소스코드(.c)를 오브젝트 파일(.o)로 변환<br>
  링크 → 여러 오브젝트 파일과 라이브러리를 결합하여 하나의 실행파일을 생성
</details>
