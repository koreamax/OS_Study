# Chapter 01. 컴퓨터 구조 시작하기

## 01-1 컴퓨터 구조를 알아야 하는 이유

좋은 개발자가 되기 위해서는 프로그래밍 언어의 문법뿐만 아니라 **컴퓨터의 근간**(컴퓨터 구조, 운영체제)을 이해해야 한다. 컴퓨터 구조를 알아야 하는 이유는 크게 두 가지다.

### 1. 문제 해결

개발 환경에서는 정상 동작하던 코드가 실제 사용자 환경에서는 동작하지 않는 경우가 많다. 이때 원인은 문법 오류가 아니라 그 너머의 근원적인 문제인 경우가 대부분이다.

- 컴퓨터를 **미지의 대상**으로 인식하면 문제 앞에서 손을 놓게 된다.
- 컴퓨터를 **분석의 대상**으로 인식하면 내부를 들여다보며 원인을 파악할 수 있다.

컴퓨터 구조를 학습하면 컴퓨터를 미지의 대상에서 분석의 대상으로 바라보는 능력, 즉 **문제 해결 능력**을 기를 수 있다. 실제로 다수의 IT 기업 채용 공고에서 컴퓨터 구조에 대한 이해를 요구하거나 필수 수강 과목으로 명시한다.

### 2. 성능, 용량, 비용

컴퓨터 구조를 학습하면 **성능, 용량, 비용**을 고려한 개발을 할 수 있다. 큰 규모의 IT 기업 중 이 세 가지를 고려하지 않고 개발하는 곳은 사실상 없다.

예를 들어 웹사이트를 운영하려면 이를 실행할 서버 컴퓨터가 필요하다. 서버 컴퓨터는 가격, 성능, 용량이 천차만별이다.

- 무조건 저렴한 컴퓨터 → 성능이 부족해 문제가 생길 수 있다.
- 무조건 최신 컴퓨터 → 불필요하게 과도한 비용을 지불할 수 있다.

따라서 내가 개발한 코드가 어느 정도의 성능과 용량을 갖춘 컴퓨터를 필요로 하는지 개발자 스스로 판단할 수 있어야 한다.

클라우드 서비스를 이용하더라도 마찬가지다. CPU 종류, 메모리, 스토리지 용량을 직접 선택해야 하며, 선택에 따라 성능과 용량은 물론 **비용**이 달라진다. 컴퓨터 구조는 결국 컴퓨터를 이루는 부품들의 성능·용량·비용에 대한 이야기라고 해도 과언이 아닐 만큼 이 주제를 자주 다룬다.

## 01-2 컴퓨터 구조의 큰 그림

본격적으로 학습하기 전에, 앞으로 배울 내용의 전체 지도를 그려 본다. 이 절에서 등장하는 용어는 이후 강의에서 반복적으로 나오므로 지금 당장 암기할 필요는 없다.

컴퓨터 구조에서 배울 내용은 크게 두 가지다.

1. 컴퓨터가 이해하는 정보
2. 컴퓨터의 4가지 핵심 부품

### 컴퓨터가 이해하는 정보

컴퓨터가 이해하는 정보에는 **데이터**와 **명령어** 두 가지가 있다.

#### 데이터

- 숫자, 문자, 이미지, 동영상과 같은 **정적인 정보**.
- 일상적으로는 컴퓨터가 주고받는 정보 또는 컴퓨터 내부에 저장된 정보를 통틀어 데이터라 부르기도 한다.

컴퓨터는 기본적으로 **0과 1**밖에 이해하지 못한다. 따라서 데이터를 다룰 때는 다음 두 가지를 학습한다.

- 0과 1로 숫자를 표현하는 방법
- 0과 1로 문자를 표현하는 방법

#### 명령어

- 컴퓨터를 실질적으로 움직이는 정보. 컴퓨터는 한마디로 **명령어를 처리하는 기계**라고 정의할 수 있다.
- 데이터는 명령어를 실행하기 위한 **재료**에 해당한다.

> **예시**
> - `1과 2를 더하라` → 명령어 / `1`, `2` → 데이터
> - `"안녕하세요"를 화면에 출력하라` → 명령어 / `"안녕하세요"` → 데이터

### 컴퓨터의 4가지 핵심 부품

라즈베리파이, 아두이노, 스마트폰, 노트북, 서버 컴퓨터 등 종류와 관계없이 컴퓨터를 동작시키는 핵심 부품은 거의 동일하다.

| 부품 | 역할 |
| --- | --- |
| **CPU** | 메모리에 저장된 명령어를 읽어 들이고 해석하고 실행한다 |
| **메모리(주기억장치)** | 현재 실행되는 프로그램의 명령어와 데이터를 저장한다 |
| **보조기억장치** | 전원이 꺼져도 보관할 프로그램을 저장한다 |
| **입출력장치** | 컴퓨터 외부와 정보를 주고받는다 |

이 부품들은 **메인보드(마더보드)** 위에 연결되며, **시스템 버스**라는 통로를 통해 서로 정보를 주고받는다.

#### 메모리

메모리는 **현재 실행되는 프로그램의 명령어와 데이터를 저장하는 부품**이다. (현재 실행되는 프로그램을 *프로세스*라고 하며, 이는 운영체제에서 다시 다룬다.)

- '메모리'는 보통 **주기억장치(Main Memory)**의 줄임말이며, 하드웨어로는 **RAM**과 **ROM**이 있다.
- 이 책에서 메모리는 일반적으로 **RAM**을 지칭한다.

**메모리의 주소**

메모리는 저장 공간마다 **주소**라는 개념을 갖는다. 메모리에는 수많은 명령어와 데이터가 저장되므로, 원하는 값이 어디에 있는지 그 위치를 식별하기 위해 주소를 사용한다.

<img width="1122" height="1402" alt="image" src="https://github.com/user-attachments/assets/939ceb52-856e-4596-967f-ee059f8d58ee" />


**메모리 핵심 3가지**

1. 프로그램이 실행되려면 메모리에 저장되어 있어야 한다.
2. 메모리는 실행되는 프로그램의 명령어와 데이터를 저장한다.
3. 메모리에 저장된 값의 위치는 **주소**로 알 수 있다.

---

#### CPU

CPU는 컴퓨터의 **두뇌**에 해당하는 부품으로, 메모리에 저장된 명령어를 **읽어 들이고, 해석하고, 실행**한다. 설계는 매우 정교하지만, 대부분의 전공서가 소개하는 핵심 내부 구성 부품은 세 가지다.

| 구성 부품 | 역할 |
| --- | --- |
| **ALU(산술논리연산장치)** | 계산을 담당하는 회로. 한마디로 계산기 |
| **레지스터** | CPU 내부의 작은 임시 저장장치. 여러 개 존재 |
| **제어장치(Control Unit)** | 제어 신호를 내보내고 명령어를 해석한다 |

**제어 신호**는 컴퓨터의 부품을 관리하고 작동시키기 위한 전기 신호다. 대표적으로 **메모리 읽기 신호**(메모리의 값을 읽고 싶을 때)와 **메모리 쓰기 신호**(메모리에 값을 저장하고 싶을 때)가 있다.

**명령어 실행 흐름 예시**

메모리 1번지의 `더하라 3번지와 4번지` 명령어를 CPU가 실행하는 과정은 다음과 같다. (이 과정은 외울 필요 없이 흐름만 이해하면 된다.)

1. 제어장치가 메모리를 향해 **메모리 읽기 신호**를 보내 1번지의 명령어를 레지스터로 가져온다.
2. 제어장치가 명령어를 해석한다 → "3번지와 4번지의 데이터가 필요하다"고 판단한다.
3. 다시 메모리 읽기 신호로 3번지(120)와 4번지(100)의 데이터를 레지스터로 가져온다.
4. ALU가 120과 100을 더해 결과 220을 레지스터에 저장한다.
5. 다음 명령어(2번지 `저장하라`)를 읽어 해석한 뒤, **메모리 쓰기 신호**와 함께 결과 220을 메모리에 저장한다.

**CPU 핵심 정리**

- CPU는 메모리에 저장된 값을 읽어 들이고, 해석하고, 실행한다.
- 내부에는 ALU(계산기), 레지스터(임시 저장장치), 제어장치(제어 신호 발생·명령어 해석)가 있다.
- 실제로는 캐시 메모리 등 더 많은 부품을 포함한다.

---

#### 보조기억장치

메모리(RAM)는 보조기억장치보다 값이 비싸고, 무엇보다 **전원이 꺼지면 저장된 내용을 잃는다**(휘발성 저장장치). 작업 중 컴퓨터가 갑자기 꺼지면 작업 내역을 잃는 이유가 여기에 있다.

따라서 전원이 꺼져도 정보를 보관할 수 있는 저장장치가 필요하며, 이것이 **보조기억장치**다. (예: USB, SD 카드, CD-ROM, 하드디스크, SSD)

- **메모리**: 실행할 프로그램을 저장한다.
- **보조기억장치**: 보관할 프로그램을 저장한다.

---

#### 입출력장치

**입출력장치**는 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 주고받는 부품이다. (예: 키보드, 마우스, 마이크, 스피커, 모니터)

보조기억장치도 외부에 연결되어 정보를 주고받으므로, 보조기억장치와 입출력장치는 딱 잘라 구분되는 개념은 아니다. 둘을 합쳐 **주변장치**라고 통칭하기도 한다.

- 다만 보조기억장치는 **메모리를 보조한다**는 특별한 임무를 수행하는 입출력장치로 볼 수 있어, 대부분의 전공서에서 구분해 서술한다.

### 메인보드와 시스템 버스

**메인보드(마더보드)**는 컴퓨터의 핵심 부품을 연결하는 판이다. 부품들은 메인보드의 슬롯에 연결되며, **버스**라는 통로를 통해 정보를 주고받는다.

버스에는 여러 종류가 있는데, 그중 핵심 부품들을 연결하는 가장 중요한(척추 역할의) 버스를 **시스템 버스**라고 한다.

시스템 버스는 세 가지 버스로 구성된다.

| 버스 | 주고받는 정보 |
| --- | --- |
| **주소 버스** | 주소 |
| **데이터 버스** | 명령어와 데이터 |
| **제어 버스** | 제어 신호 |

**예시 — 메모리 읽기**

1번지의 값을 읽는 경우:
- 제어 버스로 **메모리 읽기 신호**를 보낸다.
- 주소 버스로 읽고자 하는 **주소(1번지)**를 보낸다.
- 메모리가 두 정보를 받아 1번지의 값을 **데이터 버스**를 통해 CPU(레지스터)로 전달한다.

**예시 — 메모리 쓰기**

5번지에 220을 저장하는 경우 세 가지 정보가 필요하다.
- 주소 버스 → 저장할 위치(5번지)
- 제어 버스 → 메모리 쓰기 신호
- 데이터 버스 → 저장할 값(220)
