## Chapter 08. 입출력장치
### 08-1 장치 컨트롤러와 장치 드라이버
- 장치 컨트롤러
- 장치 드라이버

### 08-2 다양한 입출력 방법
- 프로그램 입출력
- 인터럽트 기반 입출력
- DMA 입출력

---

**최민경**

1-1. cpu가 장치 컨트롤러의 상태를 확인할 때 폴링 방식과 인터럽트 방식을 설명하고, 어떤 장단점이 있는지 말해주세요.
<details>
  <summary>정답</summary>
  폴링은 CPU가 주기적으로 장치 컨트롤러의 상태 레지스터를 직접 읽어서 작업이 완료되거나 준비되었는지 무한 반복하며 확인하는 방식입니다. 구현이 매우 단순하고 직관적이지만, cpu 자원 낭비가 심합니다. 인터럽트는 장치 컨트롤러가 작업을 끝내면 인터럽트를 보내 CPU에게 알리는 방식입니다. CPU 효율과 시스템 전체 처리량이 늘어나지만 컨텍스트 스위칭 오버헤드가 발생한다는 단점이 있습니다.
</details>

1-2. PIC를 활용한 입출력 처리 과정을 설명해주세요. 
<details> 
  <summary>정답</summary>
  PIC가 장치 컨트롤러에서 인터럽트 요청 신호를 받습니다. PIC는 우선순위를 판단하고, CPU에 인터럽트 요청 신호를 보냅니다. CPU는 PIC에 인터럽트 확인 신호를 보내고, PIC는 데이터 버스를 통해 CPU에 인터럽트 벡터를 보냅니다. CPU는 인터럽트 벡터로 인터럽트 주체를 알고, 그 장치의 인터럽트 서비스 루틴을 실행합니다.
</details>

2-1. DMA의 입출력 과정을 설명해주세요.
<details>
  <summary>정답</summary>
  CPU는 DMA 컨트롤러에 입출력 장치의 주소, 수행할 연산, 읽거나 쓸 메모리의 주소와 같은 정보로 입출력 작업을 명령합니다. DMA는 CPU 대신 장치 컨트롤러와 상호작용하며 입출력을 수행합니다. 필요한 경우 DMA는 시스템 버스로 메모리에 직접 접근하여 정보를 읽거나 씁니다. 입출력이 끝나면 DMA는 CPU에 인터럽트를 걸어 작업이 끝났음을 알립니다.
</details>

2-2. DMA는 시스템 버스가 CPU를 쓰지 않는지 어떻게 알까요?
<details>
  <summary>정답</summary>
  DMA가 메모리에 접근해야 하면 CPU에게 전기 신호를 보냅니다. CPU는 현재 실행 중인 기계어 명령 주기나 버스 사이클이 끝나는 시점을 확인하고, 버스를 넘겨줄 수 있는 상태가 되면 자신이 수행하던 버스 동작을 중단하고 시스템 버스를 고저항 상태로 만듭니다. CPU는 DMA 컨트롤러에 버스 승인 신호를 보냅니다.
</details>



**방혜윤**

1-1. 프로그램 입출력 방식을 설명해주세요
<details>
  <summary>정답</summary>
  프로그램 속 명령어로 입출력장치를 제어하는 방법으로 CPU가 프로그램 속 명령어를 실행하는 과정에서 입출력 명령어를 만나면 CPU는 입출력장치에 연결된 장치 컨트롤러와 상호작용하며 입출력 작업을 수행하는 방식입니다.
</details>

1-2. 프로그램 입출력 방식 중 메모리 맵 입출력과 고립형 입출력을 설명해주세요
<details>
  <summary>정답</summary>
  메모리 맵 입출력은 메모리 주소 공간과 입출력장치 주소 공간을 하나의 주소 공간으로 사용하는 방식입니다.<br>
  따라서 CPU는 메모리에 접근하는 명령어만으로도 장치 컨트롤러의 레지스터에 접근할 수 있습니다.<br>
  반면 고립형 입출력은 메모리 주소 공간과 입출력장치 주소 공간을 분리하여 사용하는 방식입니다. <br>
  이 경우 CPU는 메모리 접근 명령어와 별도의 입출력 명령어를 사용하여 장치 컨트롤러에 접근합니다.
</details>

2-1. 인터럽트 기반 입출력 방식에서 여러 입출력장치에서 인터럽트가 동시에 발생한 경우 인터럽트를 처리하는 방법을 설명해주세요
<details>
  <summary>정답</summary>
  다중 인터럽트가 발생하면 두 가지 방식으로 처리할 수 있습니다.<br>
  첫 번째는 발생한 순서대로 처리하는 방식입니다.<br>
  CPU가 인터럽트 처리 중 인터럽트 비트를 비활성화하면 다른 인터럽트를 받아들이지 않으므로 현재 인터럽트를 모두 처리한 뒤 다음 인터럽트를 처리합니다.<br>
  두 번째는 우선순위를 고려하는 방식입니다.<br>
  인터럽트 비트가 활성화되어 있거나 NMI와 같은 높은 우선순위의 인터럽트가 발생하면 CPU는 우선순위가 높은 인터럽트부터 처리합니다.
</details>

2-2. PIC가 CPU에 인터럽트 벡터를 전달할 때는 주소 버스가 아닌 데이터 버스를 사용하는 이유를 설명해주세요
<details>
  <summary>정답</summary>
  인터럽트 벡터가 메모리 주소 자체가 아니라 하나의 데이터(식별자) 이기 때문입니다.<br>
  CPU는 PIC로부터 받은 인터럽트 벡터를 이용해 인터럽트 벡터 테이블에서 해당 인터럽트 서비스 루틴의 시작 주소를 찾습니다.
</details>

**이민형**

1-1. CPU가 메모리에 저장된 정보를 하드디스크에 백업할 때의 흐름에 대해 설명해주세요.(DMA가 없을 때)
<details>
  <summary>정답</summary>
  먼저 CPU가 하드디스크 컨트롤러의 제어 레지스터에 쓰기 명령을 보내면, 하드디스크는 준비가 끝났을 때 상태 레지스터에 준비 완료를 표시합니다. <br>
  CPU는 이 상태 레지스터를 주기적으로 읽으며 준비 여부를 확인하고, 준비가 확인되면 메모리에서 데이터를 자신의 레지스터로 읽어온 뒤 이를 컨트롤러의 데이터 레지스터에 씁니다. <br>
  이 과정은 백업할 데이터를 모두 옮길 때까지 반복됩니다.
</details>

1-2. 상태 레지스터를 주기적으로 확인하는 방식을 뭐라고 하나요?
<details>
  <summary>정답</summary>
  폴링이라고 합니다. CPU가 장치 컨트롤러의 상태 레지스터를 반복적으로 읽어 장치의 준비나 작업 완료 여부를 직접 확인하는 방식인데, <br>
  확인하는 동안 CPU 자원을 계속 소모하기 때문에 효율이 떨어진다는 단점이 있습니다.
</details>

2-1. CPU가 메모리에 저장된 정보를 하드디스크에 백업할 때의 흐름에 대해 설명해주세요.(DMA가 있을 때)
<details>
  <summary>정답</summary>
  CPU가 DMA 컨트롤러에 연산 종류와 메모리 주소, 데이터 크기 등을 담아 입출력 작업을 명령하면, 이후의 실제 전송은 DMA 컨트롤러가 CPU를 대신해 수행합니다. <br>
  DMA 컨트롤러는 메모리에서 데이터를 읽어 하드디스크의 장치 컨트롤러로 직접 내보냅니다. <br>
  전송이 모두 끝나면 DMA 컨트롤러가 CPU에 인터럽트 신호를 보내 작업이 완료되었음을 알립니다.
</details>

2-2. 사이클 스틸링의 문제와 이를 해결하기 위한 방안에 대해서 설명해주세요.
<details>
  <summary>정답</summary>
  사이클 스틸링의 문제는 시스템 버스가 동시에 사용될 수 없다는 점에서 비롯됩니다. <br>
  DMA 컨트롤러가 메모리에 접근하기 위해 버스를 사용하는 동안에는 CPU가 그 버스를 쓸 수 없으므로, 사이클 스틸링이 자주 일어날수록 CPU가 대기하는 시간이 늘어 성능 손해가 커집니다. <br>
  이를 줄이기 위한 방안은 입출력 전용 통로인 입출력 버스를 따로 두는 것입니다. <br>
  PCI 버스나 PCIe 버스, PCIe 슬롯과 같은 입출력 버스를 통해 입출력 관련 통신을 시스템 버스에서 분리하면, 시스템 버스를 두고 벌어지는 경쟁이 줄어들어 사이클 스틸링으로 인한 손해를 완화할 수 있습니다.
</details>
