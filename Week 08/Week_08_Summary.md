# Chapter 07. 보조기억장치

RAM(메모리)은 휘발성이라 전원이 꺼지면 내용을 잃는다. 전원이 공급되지 않아도 내용을 유지하는 저장 장치가 **보조기억장치**다. 이 장에서는 대표적인 보조기억장치인 **하드 디스크**와 **플래시 메모리**, 그리고 이들을 효율적으로 사용하는 **RAID**를 다룬다.

## 07-1 다양한 보조기억장치

### 하드 디스크

**하드 디스크**는 **자기적인 방식**으로 데이터를 저장하는 보조기억장치로, 자기 디스크의 일종이다. (LP판과 비슷하게 동작한다.)

#### 하드 디스크의 구성

| 구성 요소 | 역할 |
| --- | --- |
| **플래터(platter)** | 수많은 N극·S극(자기적 물질)으로 데이터를 저장하는 원판. 보통 여러 겹을 쓰고 양면을 모두 사용 |
| **스핀들(spindle)** | 플래터를 회전시키는 구성 요소. 회전 속도 단위는 **RPM**(분당 회전 수) |
| **헤드(head)** | 플래터에 미세하게 떠서 자기적 물질을 읽고 쓰는 구성 요소. 각 면마다 존재 |
| **디스크 암(disk arm)** | 헤드를 움직이는 구성 요소. 보통 모든 헤드가 디스크 암에 부착되어 함께 이동 |

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/af6b6a84-617d-4768-bf0c-22fec40a09ea" />


#### 하드 디스크의 저장 단위

| 단위 | 설명 |
| --- | --- |
| **트랙(track)** | 플래터를 이루는 동심원 (운동장 트랙처럼) |
| **섹터(sector)** | 트랙을 일정하게 나눈 한 조각. 데이터 저장의 기본 단위 |
| **블록(block)** | 하나 이상의 섹터를 묶은 단위 (운영체제 파일 시스템에서 다시 등장) |
| **실린더(cylinder)** | 여러 겹의 플래터에서 같은 위치의 트랙을 모아 연결한 원통 모양의 논리적 단위 |

> **연속된 정보는 한 실린더에 기록된다.** 헤드는 디스크 암에 부착되어 함께 움직이므로, 같은 실린더(같은 트랙 위치)에 저장하면 헤드를 움직이지 않고 바로 읽을 수 있기 때문이다.

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/3313f3c3-62a0-4124-ae7b-9f3ea94fed2e" />


#### 하드 디스크의 데이터 접근 과정

하드 디스크에 저장된 값을 읽거나 쓸 때는 보통 세 가지 시간이 소요된다.

| 시간 | 설명 |
| --- | --- |
| **탐색 시간(seek time)** | 데이터가 저장된 트랙까지 **헤드를 이동**시키는 시간 |
| **회전 지연(rotational latency)** | 원하는 데이터까지 **플래터를 회전**시키는 시간 |
| **전송 시간(transfer time)** | 하드 디스크와 컴퓨터 간에 데이터를 **실제로 전송**하는 시간 |

> 이 시간들은 CPU·메모리의 동작 시간에 비해 매우 길어, 성능에 중요한 변인이 된다. (제프 딘의 "모든 프로그래머가 알아야 할 숫자들"에서도, 하드 디스크 탐색 시간은 L1·L2 캐시나 메모리 참조 시간보다 압도적으로 크게 제시된다.)

### 플래시 메모리

**플래시 메모리**는 **전기적으로** 데이터를 읽고 쓰는 반도체 기반 저장 장치다. SSD, SD 카드, USB 메모리가 모두 플래시 메모리 기반이다. (범용성이 넓어 주기억장치인 ROM으로도 쓰이지만, 보조기억장치로도 많이 사용된다.)

플래시 메모리에는 **낸드(NAND) 플래시**와 **노어(NOR) 플래시**가 있으며, 오늘날 대용량 저장 장치로 많이 쓰이는 것은 **낸드 플래시 메모리**다.

#### 셀과 비트 (SLC, MLC, TLC, QLC)

**셀(cell)**은 플래시 메모리에서 데이터를 저장하는 가장 작은 단위다. 한 셀에 몇 비트를 저장하느냐에 따라 종류가 나뉘며, 이는 성능·가격·수명을 좌우한다.

> **집/사람 비유** — 셀을 집, 1비트를 사람 한 명으로 보면: 혼자 살수록(SLC) 외출(입출력)이 빠르고, 여럿이 살수록(TLC) 외출이 느리지만 한 집에 더 많이 살 수 있어(대용량) 1인당 비용이 저렴하다.

| 타입 | 셀당 비트 | 표현 가능 정보 | 입출력 속도 | 수명 | 용량 대비 가격 |
| --- | --- | --- | --- | --- | --- |
| **SLC** (Single Level Cell) | 1비트 | 2가지 | 빠름 | 김 | 비쌈 |
| **MLC** (Multiple Level Cell) | 2비트 | 4가지 | 중간 | 보통 | 보통 |
| **TLC** (Triple Level Cell) | 3비트 | 8가지 | 느림 | 짧음 | 저렴 |
| **QLC** (Quadruple Level Cell) | 4비트 | 16가지 | 더 느림 | 더 짧음 | 더 저렴 |

> 셀당 비트가 많을수록 대용량화에 유리하지만 입출력 속도와 수명은 떨어진다. 시중의 USB·SSD는 주로 MLC·QLC 계열이 많이 쓰인다. (플래시 메모리와 하드 디스크 모두 **수명**이 있어, 일정 횟수 이상 쓰기/지우기를 반복하면 수명을 다한다.)

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/51f241b7-cf4d-4b51-92d9-061e433ebd07" />


#### 플래시 메모리의 저장 단위

셀이 모여 점점 큰 단위를 이룬다.

```
셀(cell) → 페이지(page) → 블록(block) → 플레인(plane) → 다이(die)
```

플래시 메모리의 가장 큰 특성은 **읽기·쓰기 단위와 삭제 단위가 다르다**는 점이다.

- **읽기·쓰기**: **페이지** 단위
- **삭제**: 페이지보다 큰 **블록** 단위

**페이지의 상태** — 플래시 메모리는 하드 디스크와 달리 **덮어쓰기가 불가능**하다. 그래서 페이지는 세 가지 상태를 가진다.

| 상태 | 의미 |
| --- | --- |
| **Free** | 어떤 데이터도 저장하지 않아 새로 저장할 수 있는 상태 |
| **Valid** | 유효한 데이터를 저장하고 있는 상태 |
| **Invalid** | 유효하지 않은 데이터(쓰레기 값)를 저장하고 있는 상태 |

#### 가비지 컬렉션

덮어쓰기가 불가능해, 기존 페이지(A)를 수정하려면 그 페이지를 **Invalid**로 만들고 새 페이지에 값을 쓴다. 이런 Invalid 페이지가 쌓이면 용량이 낭비된다.

**가비지 컬렉션(garbage collection)**은 유효한(Valid) 페이지들만 새 블록으로 복사한 뒤 기존 블록을 통째로 삭제해, 저장 공간을 정리하는 기법이다.

## 07-2 RAID의 정의와 종류

### RAID의 정의

**RAID(Redundant Array of Independent Disks)**는 독립적인 보조기억장치(하드 디스크·SSD) 여러 개를 마치 하나의 논리적 장치처럼 엮어 사용하는 기술이다. 데이터의 **안전성**이나 높은 **성능**을 위해 사용한다.

> 예) 1TB 하드 디스크 4개로 RAID를 구성하면, 4TB 하드 디스크 1개의 성능·안전성을 능가할 수 있다.

RAID를 구성하는 방법(종류)을 **RAID 레벨**이라 하며, RAID 0, 1, 2, 3, 4, 5, 6과 이들을 조합한 **네스티드 RAID**가 있다. 아래에서는 많이 쓰이는 0, 1, 4, 5, 6을 다룬다.

> **스트라이핑(striping)** — 데이터를 여러 디스크에 줄무늬처럼 분산해 저장하는 것. 분산 저장된 데이터 조각을 **스트라이프(stripe)**라고 한다.

### RAID의 종류

#### RAID 0

데이터를 여러 디스크에 단순히 **나누어 분산 저장**(스트라이핑)하는 방식이다.

- **장점**: 여러 디스크에서 동시에 입출력할 수 있어 **입출력 속도가 향상**된다.
- **단점**: 복사본·오류 복구 정보가 없어, 디스크 하나만 고장 나도 전체 데이터를 사용할 수 없다. → **안전하지 않다.**

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/c0b8c49e-baaf-46d7-8b29-30bd4b6255db" />


#### RAID 1 (미러링)

거울(mirror)처럼 **완전한 복사본**을 만드는 방식이다.

- **장점**: 원본이 손상되어도 복사본이 있어 **백업·복구가 쉽고 안전하다.**
- **단점**: 같은 데이터를 두 곳에 써야 해 **쓰기 속도가 느릴 수** 있고, 복사본 용량만큼 사용 가능한 용량이 줄어 **많은 디스크가 필요**(비용 증가)하다.

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/afb2c841-a982-4b94-a8cf-6e30c7c46cab" />


#### RAID 4

완전한 복사본 대신, 오류를 검출·복구하기 위한 정보인 **패리티 비트**를 **별도의 디스크 하나**에 모아 저장하는 방식이다.

- RAID 1보다 적은 디스크로 안전하게 데이터를 보관할 수 있다.
- **단점**: 데이터를 쓸 때마다 패리티 디스크에도 써야 하므로, **패리티 디스크에 병목 현상**이 생길 수 있다.

> RAID에서 쓰이는 패리티 비트는 일반적인 오류 **검출**용을 넘어 오류 **복구**까지 가능하다.

#### RAID 5

패리티 정보를 한 디스크에 몰아두지 않고 **여러 디스크에 분산 저장**하는 방식이다. RAID 4의 패리티 디스크 **병목 현상을 완화**한다.

<img width="1448" height="1086" alt="image" src="https://github.com/user-attachments/assets/ddc2e910-d299-4c35-9c44-a9489442905e" />


#### RAID 6

**두 종류의 패리티**(오류 검출·복구 수단)를 사용하는 방식이다.

- RAID 5보다 **더 안전**하다.
- **단점**: 패리티를 두 종류 써야 하므로 **쓰기 속도가 더 느릴 수** 있다.

### RAID 정리

| 레벨 | 방식 | 특징 |
| --- | --- | --- |
| **RAID 0** | 분산 저장(스트라이핑) | 입출력 속도↑, 안전성↓ |
| **RAID 1** | 완전한 복사본(미러링) | 안전성↑, 가용 용량↓·비용↑ |
| **RAID 4** | 패리티를 전용 디스크에 저장 | 적은 디스크로 안전, 패리티 디스크 병목 |
| **RAID 5** | 패리티를 분산 저장 | RAID 4의 병목 완화 |
| **RAID 6** | 두 종류의 패리티 사용 | RAID 5보다 안전, 쓰기 속도↓ |

> 최우선 목표(속도/안전성)에 따라 최적의 RAID 레벨이 달라진다. 빠른 입출력이 중요하면 RAID 0, 안전성이 중요하면 RAID 1이나 6이 유리할 수 있다. 실제로 BIOS 셋업 등에서 RAID 레벨과 스트라이프 크기를 직접 설정할 수 있다.
